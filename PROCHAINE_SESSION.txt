CHANTIER 1 — TRANSCRIPTION AUDIO (PRIORITÉ)
================================================================

OBJECTIF :
Quand un prospect envoie un message vocal WhatsApp,
le bot doit le transcrire via Groq Whisper puis traiter
le texte comme un message normal.

CONTEXTE TECHNIQUE :
- Actuellement dans messageHandler.js : les messages non-texte
  reçoivent un message d'erreur générique ("je ne traite que le texte")
- WhatsApp envoie les audios avec type: "audio" et un media_id
- Il faut : 1) télécharger le fichier audio via l'API Meta
            2) envoyer à Groq Whisper pour transcription
            3) traiter le texte transcrit comme un message normal

API GROQ :
- Site : https://console.groq.com
- Modèle Whisper : whisper-large-v3 (le meilleur) ou whisper-large-v3-turbo
- Gratuit avec limites généreuses
- SDK : npm install groq-sdk
- Doc : https://console.groq.com/docs/speech-text

ÉTAPES D'IMPLÉMENTATION :
1. Créer un compte Groq sur console.groq.com → récupérer la clé API
2. Ajouter GROQ_API_KEY dans .env et dans Railway
3. Créer src/transcription.js :
   - downloadAudioFromMeta(mediaId) → télécharge le fichier audio
   - transcribeWithGroq(audioBuffer) → retourne le texte
4. Modifier src/messageHandler.js :
   - Si type === 'audio' → transcribeWithGroq() → traiter comme texte
   - Loguer "[vocal transcrit]" dans Google Sheets
5. Modifier src/whatsapp.js :
   - Ajouter getMediaUrl(mediaId) pour récupérer l'URL de téléchargement
   - Ajouter downloadMedia(mediaUrl) pour télécharger le fichier

VARIABLE À AJOUTER dans .env :
GROQ_API_KEY=           ← clé récupérée sur console.groq.com

CHANTIER 3 — FICHIER DE LEADS
================================================================

OBJECTIF :
Permettre d'envoyer automatiquement le MSG 1 (premier contact)
à une liste de prospects depuis un fichier de leads,
sans avoir à envoyer manuellement chaque message.

FORMAT ATTENDU DU FICHIER :
CSV ou Google Sheet avec colonnes :
| Prénom | Numéro (format international sans +) | Statut |

FONCTIONNEMENT ATTENDU :
1. Script ou endpoint qui lit la liste
2. Pour chaque lead avec statut "à contacter" :
   - Envoie le MSG 1 personnalisé avec le prénom
   - Marque le statut "envoyé" dans le fichier
   - Délai entre chaque envoi (éviter le spam Meta)
3. Les réponses sont traitées normalement par le webhook existant

FICHIERS À CRÉER OU MODIFIER :
- src/leads.js → nouveau fichier : lecture fichier leads + envoi MSG 1
- Ou une nouvelle route dans server.js : POST /launch-campaign
- Ou un script standalone : scripts/sendLeads.js

QUESTION EN SUSPENS :
Format du fichier leads → CSV local, Google Sheet ou autre ?
À clarifier au début de la session.
