CHANTIER 3 — CHATWOOT (À FAIRE)
================================================================

OBJECTIF : Interface CRM pour lire et gérer toutes les conversations.
           Pause/reprise automatique quand un humain prend la main.

ÉTAPES :
1. Déployer Chatwoot sur Railway (instance séparée)
   → Image Docker officielle : chatwoot/chatwoot
   → Base de données PostgreSQL Railway
2. Créer 2 boîtes de réception dans Chatwoot :
   - "Agent Qualification" → leads froids
   - "Agent Nurturing" → membres Skool
3. Créer src/chatwoot.js → API Chatwoot (créer/mettre à jour conversations)
4. Modifier messageHandler.js → envoyer chaque échange à Chatwoot
5. Logique pause/reprise :
   - Conversation assignée dans Chatwoot → bot se tait
   - Conversation remise en "non assignée" → bot reprend
   - Stocker les numéros en pause dans une liste mémoire
6. Webhook Chatwoot → POST /chatwoot-webhook dans server.js
   → Écouter les événements d'assignation/désassignation

VARIABLES À AJOUTER :
CHATWOOT_URL=
CHATWOOT_API_TOKEN=
CHATWOOT_INBOX_ID_QUALIFICATION=
CHATWOOT_INBOX_ID_NURTURING=
